# Getting Help

Thanks for using Triggery! Please pick the channel that best matches your need.

## I have a question about usage

→ Open a [GitHub Discussion in **Q&A**](https://github.com/triggeryjs/triggery/discussions/new?category=q-a).

Searchable, indexed by Google, and the answer helps the next person too.
Stack Overflow questions are welcome under the
[`triggery` tag](https://stackoverflow.com/questions/tagged/triggery) once the
project gains traction.

## I think I found a bug

→ Open a [Bug report issue](https://github.com/triggeryjs/triggery/issues/new?template=bug_report.yml).

Please include:

- Triggery version (`@triggery/core` and any binding/adapter).
- Framework version (React 18/19, Solid 1.x, Vue 3.x).
- A minimal reproduction (StackBlitz/CodeSandbox link strongly preferred).

## I have an idea for a feature

→ Open a [Feature request issue](https://github.com/triggeryjs/triggery/issues/new?template=feature_request.yml)
for small additions, or
[Show and tell / Ideas discussion](https://github.com/triggeryjs/triggery/discussions/new?category=ideas)
for larger directional changes.

For sweeping API changes, please use the [**RFC** template](https://github.com/triggeryjs/triggery/issues/new?template=rfc.yml)
so the proposal can be reviewed against the project's stability guarantees.

## I want to chat

→ Join the [**Triggery Discord**](https://triggeryjs.github.io/discord/) — live
help in `#help-react` / `#help-solid` / `#help-vue` / `#help-adapters` /
`#help-devtools`, RFC discussions in `#rfc-proposals`, project notes in
`#devlog`.

For anything you'd want a future visitor to find via Google,
[GitHub Discussions](https://github.com/triggeryjs/triggery/discussions) is
still the better channel — chat scroll loses things, indexed Q&A doesn't.
