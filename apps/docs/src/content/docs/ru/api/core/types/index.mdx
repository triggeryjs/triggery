## Остальные экспортируемые типы

Стабильные, но описаны инлайн в исходниках, без отдельной страницы. Открой ссылку чтобы прыгнуть прямо к декларации.

| Тип | Где |
|---|---|
| `Runtime` | [`types.ts`](https://github.com/triggeryjs/triggery/blob/main/packages/core/src/types.ts) — экземпляр рантайма который возвращает `createRuntime` |
| `Trigger<S>` | [`types.ts`](https://github.com/triggeryjs/triggery/blob/main/packages/core/src/types.ts) — значение, возвращаемое `createTrigger` |
| `TriggerHandler<S, R>` | [`types.ts`](https://github.com/triggeryjs/triggery/blob/main/packages/core/src/types.ts) — сигнатура `config.handler` |
| `ConcurrencyStrategy` | [`types.ts`](https://github.com/triggeryjs/triggery/blob/main/packages/core/src/types.ts) — `'take-latest' \| 'take-every' \| 'take-first' \| 'queue' \| 'exhaust' \| 'sync'` |
| `SchedulerStrategy` | [`types.ts`](https://github.com/triggeryjs/triggery/blob/main/packages/core/src/types.ts) — `'microtask' \| 'sync'` |
| `CheckCtx<C>` | [`types.ts`](https://github.com/triggeryjs/triggery/blob/main/packages/core/src/types.ts) — типизированный `check.is` / `check.all` / `check.any` |
| `MetaCtx` | [`types.ts`](https://github.com/triggeryjs/triggery/blob/main/packages/core/src/types.ts) — `triggerId`, `runId`, `cascadeId`, `scheduledAt` |
| `TriggerInspectSnapshot` | [`types.ts`](https://github.com/triggeryjs/triggery/blob/main/packages/core/src/types.ts) — одна запись кольцевого буфера |
| `FireContext` / `MatchContext` / `SkipContext` / `ActionContext` / `CascadeContext` | [`types.ts`](https://github.com/triggeryjs/triggery/blob/main/packages/core/src/types.ts) — payload-ы middleware-хуков |
| `RegistrationToken` | [`types.ts`](https://github.com/triggeryjs/triggery/blob/main/packages/core/src/types.ts) — возвращается каждым `runtime.register*` |
| `EmptyRecord` | [`types.ts`](https://github.com/triggeryjs/triggery/blob/main/packages/core/src/types.ts) — `Record<string, never>` для схем без actions/conditions |
| `EventMap<S>` / `ConditionMap<S>` / `ActionMap<S>` | [`types.ts`](https://github.com/triggeryjs/triggery/blob/main/packages/core/src/types.ts) — sub-map'ы схемы |
| `EventKey<S>` / `ConditionKey<S>` / `ActionKey<S>` | [`types.ts`](https://github.com/triggeryjs/triggery/blob/main/packages/core/src/types.ts) — union имён внутри каждого sub-map |
| `NamedHooks<S>` | [`types.ts`](https://github.com/triggeryjs/triggery/blob/main/packages/core/src/types.ts) — результат `createNamedHooks` |

Если хочешь чтобы какой-то тип получил отдельную страницу — [открой issue](https://github.com/triggeryjs/triggery/issues/new), это дёшево добавить.
